# 자동화 도구 비교 구현 (Make vs Zapier)

**워크플로우:** "Google Sheets 새 행 추가 → 조건 분기(점수 기준) → Discord 알림 발송 + Gmail 전송"

### [Make 구현]
- **Trigger:** Google Sheets – Watch New Rows
- **Router:** 점수 80점 이상 / 미만 분기
- **Action 1:** Discord – Send a Message (합격 알림)
- **Action 2:** Gmail – Send an Email (합격 축하 메일)

### [Zapier 구현]
- **Trigger:** Google Sheets – New Spreadsheet Row
- **Filter:** 점수 80점 이상만 통과 (COL B Greater than 79)
- **Action 1:** Discord – Send Channel Message
- **Action 2:** Gmail – Send Email

---

### 구현 과정 요약
- **데이터 소스 정의:** Google Sheets를 트리거로 설정하여 새 행이 추가될 때마다 학생 성적 데이터(이름, 점수, 이메일)를 자동으로 감지하도록 구성함.
- **조건 분기 설정:** '점수' 필드를 기준으로 80점 이상인 경우에만 후속 작업이 실행되도록 필터링함.
- **다중 액션 실행:** 조건 만족 시 하나의 트리거로부터 Discord 채널 메시지 발송과 Gmail 축하 이메일 전송이라는 2개 이상의 서로 다른 액션이 순차적으로 실행되도록 워크플로우를 완결함.
- **도구별 제약 우회:** Make에서는 내장 기능인 Router를 사용하여 다중 분기를 직관적으로 처리하였으며, Zapier 무료 플랜에서는 다중 분기(Paths) 제약을 우회하기 위해 Filter 모듈을 도입하여 조건 불충족 데이터를 차단하는 방식으로 동일한 결과물을 도출함.

### 비교 항목

| 번호 | 비교 항목 | Make (메이크) | Zapier (재피어) |
| :--- | :--- | :--- | :--- |
| 1 | UI/UX | 시각적인 동그라미(노드) 마인드맵 형태 구조. 드래그 앤 드롭으로 복잡한 다차원 흐름을 자유롭게 조망하기 유리함. | 위에서 아래로 단방향으로 내려오는 리스트 형태 구조. 인터페이스가 정돈되어 있어 텍스트 중심 가독성이 높음. |
| 2 | 설정 난이도 | 자유도가 높은 대신 각 모듈 간의 세부 데이터 매핑 및 데이터 타입 지정 시 초기 학습 곡선이 다소 존재함. | 단계별 설정 가이드 마법사가 매우 친절하여 초보자도 마우스 클릭 몇 번으로 쉽게 연동을 마칠 수 있음. |
| 3 | 연동 서비스 범위 | 글로벌 주요 서비스 연동을 넓게 지원하나, 마이너한 앱의 경우 웹훅(Webhook)이나 직접 API 설정이 필요할 수 있음. | 현존하는 자동화 도구 중 가장 방대한 양의 서드파티 앱과 공식 커넥터를 보유하고 있어 앱 생태계 접근성이 매우 뛰어남. |
| 4 | 무료 플랜 범위 | **매월 1,000번의 실행 수(Ops)**를 기본 제공하여 복잡하고 잦은 연동 테스트 및 실무 적용 시 비용 부담이 적음. | **매월 100번의 실행(Tasks)**만 제공하며 다단계 액션 설계 시 횟수가 빠르게 차감되어 무료 플랜의 제약이 큼. |
| 5 | 실행 로그 확인 방식 | 실행 히스토리 맵에서 각 노드를 마우스로 클릭해 단계별 입력(Input) 및 출력(Output) 데이터를 시각적으로 정밀 디버깅 가능함. | Task History 메뉴에서 텍스트 기반의 단계별 성공/실패 로그 리스트를 제공하여 깔끔하게 에러 발생 지점을 파악할 수 있음. |

### 각 도구의 장단점 정리

#### 1) Make
- **장점:** 무료 플랜의 혜택(1,000 Ops)이 크고, 다중 조건 분기(Router)를 개수 제한 없이 무료로 설계할 수 있어 고차원 자동화에 유리함.
- **단점:** 초기 인터페이스 구조가 다소 복잡하여 툴에 익숙해지는 데 시간이 걸림.

#### 2) Zapier
- **장점:** 인터페이스가 극도로 직관적이며, 전 세계 거의 모든 서드파티 앱과의 커넥터를 공식 지원함.
- **단점:** 무료 플랜의 실행 횟수가 너무 적고, 다중 분기(Paths) 기능이 유료 등급에만 묶여 있어 우회 설계가 필요함.

---

### 어떤 상황에서 적합한지 의견

- **Make가 적합한 상황:** 비용을 들이지 않고 무료 플랜 내에서 복잡한 다단계 조건 분기(Router)나 대규모 데이터 매핑이 필요한 정교한 자동화 아키텍처를 구축하고자 할 때 가장 적합함.
- **Zapier가 적합한 상황:** 복잡한 기능 제약이나 학습 시간 없이, 대중적인 앱들을 빠르게 결합하여 단방향성(일직선 구조)의 직관적인 프로토타입 자동화를 초고속으로 구축하고 검증하고 싶을 때 적합함.

### [프로젝트 2] 자유 주제 자동화 설계 및 구현 (Make)

#### 1) 자동화할 반복 업무 정의
* **반복 업무:** 매일 포털 사이트나 날씨 앱에 수동으로 접속하여 현지(영국 런던)의 기상 정보를 검색하고 확인하는 일상적인 루틴.
* **자동화 목적:** 매일 지정된 시각에 영국 런던 지역의 날씨 데이터를 수집한 뒤, 날씨 상태(맑음/흐림 또는 비/눈)를 스스로 판별하여 주로 사용하는 협업 도구(Discord)와 이메일(Gmail)로 알림을 동시 전송함. 이를 통해 기상 변화에 따른 알림을 놓치지 않고 정보 탐색 시간을 최소화함.

#### 2) 도구 선정 및 선정 이유
* **선정 도구:** Make (메이크)
* **선정 이유:** 별도의 외부 날씨 API 키 발급 없이도 내장된 'Weather' 모듈로 전 세계 날씨 데이터를 즉시 호출할 수 있음. 특히 Zapier 등의 타 도구와 달리, 무료 플랜 범위(월 1,000회 실행) 내에서도 'Router'와 'Filter' 기능을 제한 없이 활용하여 복잡한 다중 조건 분기 및 다중 채널(Discord + Gmail) 알림 발송 구조를 비용 없이 완벽하게 구현할 수 있기 때문임.

#### 3) 자동 실행 구조 구현
* **자동 실행 조건:** 매일 오후 10시에 자동으로 전체 시나리오가 트리거되어 작동하도록 내부 스케줄러 세팅을 완료함. 또한, 수집된 날씨 결과에 따라 실행 경로가 다르게 흐르는 조건부 자동화 구조를 적용함.

#### 4) 워크플로우 흐름 설명
* **Trigger (스케줄 기반 날씨 데이터 호출):** 매일 오후 10시 정각이 되면 Weather - Get Current Weather 모듈이 실행되어 영국 런던 지역(London, uk)의 현재 기온 및 기상 데이터를 API 서버에서 실시간으로 호출함.
* **Router & Filter (날씨 상태별 조건 분기):** Weather 모듈 뒤에 Router를 배치하여 데이터 경로를 나눔. 데이터의 대소문자 구분 오류를 방지하기 위해 Equal to (case insensitive) 연산자를 사용하였으며, 맑거나 흐린 날씨(Clear, Clouds) 경로와 비나 눈이 오는 날씨(Rain, Snow) 경로로 흐름을 필터링하도록 세팅함.
* **Action (다중 채널 알림 직렬 전송):** 분기된 각 경로마다 Discord - Create a Message 모듈과 Gmail - Send an Email 모듈을 직렬(연속)로 연결함. 필터를 통과한 날씨 상태에 따라 맞춤형 안내 메시지가 디스코드 채널과 구글 메일로 동시에 자동 발송되도록 구현함.
